Visi & Misi

  • Visi: Membuat AI pendamping yang mudah dipakai oleh siapa saja — untuk belajar, berkarya, dan produktivitas sehari-hari.
  • Misi: Menyediakan akses AI berkualitas lewat antarmuka sederhana, fitur kolaboratif (coding partner), dan dukungan file/media analysis.

Ringkasan teknis (apa yang dibangun)

Berikut ini ringkasan fitur teknis berdasarkan kode sumber aplikasi:

  • Chat AI interaktif dengan sesi percakapan (penyimpanan sesi, manajemen sesi, dan kemampuan berbagi sesi melalui link publik).
  • Mode Coding Partner — fitur yang memfasilitasi debugging, penulisan kode, dan penyimpanan proyek-agent.
  • Unggah gambar dan analisis (image upload & analysis) serta image generation/editing (download, share, export image).
  • Text-to-Speech (TTS) — pemutaran jawaban AI dengan beberapa opsi suara (termasuk tombol pemutaran di UI).
  • Export chat ke PDF, copy/paste dan share pesan.
  • Analisis file & PDF (membaca dokumen yang diunggah dan menganalisis konten).
  • Analisis video / YouTube — kemampuan memproses dan menganalisis metadata/teks dari video.
  • Shared chats — hasil percakapan dapat dibuat link publik yang dibuka tanpa login.
  • Subscription plans & pembatasan kuota — rancangan paket (Free/Junior/Senior/Superior/Unlimited) dengan model dan batasan berbeda.
  • PWA-ready — manifest dan konfigurasi untuk instalasi sebagai aplikasi di perangkat mobile.
  • Integrasi pembayaran (Pakasir) untuk fitur berbayar.
  • Backend ringan (Supabase functions & migrations) untuk menyimpan user profiles, memory, session, dan pesan.

Pengalaman pengguna & privasi

AqualibriaAI dirancang agar mudah digunakan: antarmuka yang responsif, dukungan export, dan fitur berbagi. Sistem menyimpan data sesi dan memori AI demi pengalaman konsisten antar kunjungan. Data pribadi (email, profil) diperlukan hanya bila pengguna memilih fitur yang membutuhkan login; beberapa halaman publik, termasuk halaman ini, dapat diakses tanpa login. Halaman ini dapat diakses tanpa login dan bisa diakses dengan mudah oleh Googlebot.

Keamanan & kebijakan singkat

  • Autentikasi user menggunakan Firebase / Supabase (tergantung implementasi deploy).
  • Data sensitif harus dimasukkan hanya bila pengguna mempercayai link yang mereka gunakan.
  • Pengguna dapat menghapus data profil melalui fitur akun (jika tersedia di deploy Anda).

Instruksi singkat untuk developer & SEO

Jika Anda developer yang ingin mengintegrasikan halaman ini ke situs Vite/React:

  1. Tempatkan file ini di folder public/about.html agar dapat diakses statis tanpa perlu autentikasi.
  2. Pastikan hosting (Vercel/Netlify) tidak memblokir file di folder public dan tidak ada redirect global yang mengarahkan semua rute ke index tanpa pengecualian untuk file statis.
  3. Tambahkan tautan ke header/footer agar Googlebot menemukan halaman ini lebih mudah.
  4. Verifikasi dengan Google Search Console dan gunakan Rich Results Test untuk memastikan JSON-LD terdeteksi dengan benar.

Penjabaran Fitur Teknis (lebih mendetail)

1) Chat & Sesi

Sistem chat memiliki struktur sesi, dengan kemampuan menyimpan riwayat chat lokal dan/atau ke backend bila pengguna login. Fitur penting termasuk:

  • Simpan sesi percakapan (lokal atau backend)
  • Buat judul sesi, pin/archive, dan rename
  • Bagikan sesi melalui link publik (shared chats) — membuka tanpa login

2) Mode Coding Partner

Mode ini menyiapkan environment khusus untuk pengembangan dan debugging, termasuk fungsi menyimpan proyek (project files) dan mengirim/menerima kode di chat.

3) Gambar & Media

AqualibriaAI menyediakan:

  • Upload gambar untuk dianalisis
  • Generate dan edit gambar (download PNG, share via Web Share API)
  • Thumbnailing & canvas-toBlob fallback untuk metode download yang reliable pada berbagai browser

4) File, PDF & Video Analysis

Platform mampu membaca PDF dan file teks untuk analisis konteks. Terdapat integrasi untuk menganalisis video/YouTube (mengambil metadata atau teks untuk ringkasan).

5) Sistem Membership & Langganan

Paket layanan diatur dengan batasan kuota harian, model AI yang digunakan, dan keistimewaan seperti LatentLeaf atau agent points.

6) PWA & Offline

Site manifest dan konfigurasi PWA disertakan sehingga pengguna dapat menambahkan aplikasi ke homescreen dan mendapatkan pengalaman hampir-native (offline-first untuk beberapa resource).

7) Pembayaran

Integrasi dengan gateway Pakasir untuk pembuatan pembayaran dan pengecekan status transaksi (digunakan untuk fitur berbayar/upgrade).

8) Backend ringan

Supabase (migrations dan fungsi serverless) menyediakan tabel untuk user_profiles, ai_memory, chat_sessions, dan chat_messages.

Catatan Pengembang

Jika Anda ingin memasukkan halaman ini ke dalam alur aplikasi React sebagai route dinamis, saya dapat membantu membuat versi React (src/pages/About.tsx) yang menggunakan layout aplikasi. Untuk penempatan statis, cukup taruh file ini di folder public/ pada project Vite/React agar dapat diakses langsung oleh server hosting tanpa memerlukan autentikasi JavaScript khusus.

Cara memastikan Googlebot dapat mengakses

  1. Pastikan robots.txt tidak memblokir /about.html atau folder /public.
  2. Jangan menaruh redirect yang menuntut cookie/session sebelum menampilkan halaman statis.
  3. Tambahkan link ke halaman ini dari halaman utama (header/footer) sehingga Googlebot menemukan dan mengikuti tautan.
  4. Gunakan Google Search Console untuk meminta indexing atau pengecekan URL setelah deploy.
